# Grounded Theory Protocol

## Trigger Pattern

Use this skill when all of the following are true:

- The input is a long qualitative text such as an interview transcript, observation note, field diary, or reflective narrative.
- The user asks for deep qualitative analysis, thematic extraction, conceptual abstraction, mechanism identification, or theory building.
- The expected result is not a surface summary but an explanatory structure grounded in the data.

## Core Method

### 1. Open Coding

- Split the text into minimal meaning units.
- Label each unit with a close-to-data phrase, ideally an in-vivo expression or a concise gerund phrase.
- Do not prematurely merge codes.
- Capture detail broadly, including contradictions and marginal phenomena.

Examples:

- "学生评教分数低，但课堂互动不少" can be coded as `遭遇量化评价误读` and `教学投入与结果背离`.
- Similar incidents should still be coded separately before deciding whether they represent the same pattern.

### 2. Axial Coding

- Group the open codes into higher-order categories.
- Explain how categories connect through conditions, context, actions, interactions, and consequences.
- Build a structured category network rather than a flat theme list.

Example:

- Codes such as `教学投入边缘化` and `应付硬性指标` can be organized under a broader category like `计算型教学投入`.

### 3. Selective Coding

- Identify the single core category that best integrates the axial network.
- Write a concise but complete theoretical storyline that explains how the major categories fit together.
- Use the optional `research_focus` only to guide emphasis, not to erase emergent findings.

Example:

- A final core category may be `学术锦标赛下的教学-科研注意力策略性分配`.

## Analytic Discipline

- Use constant comparison throughout the workflow.
- Keep memo notes on mechanisms, tensions, uncertainty, and category revisions.
- Ask repeatedly: what mechanism or logic does this data pattern reveal?
- Preserve disconfirming evidence instead of forcing consensus.
- Prefer interpretation that remains visibly grounded in the original text.

## Auditability And Lossless Processing

- The workflow must preserve the full original corpus as a run artifact before any transformation.
- Node A must output a numbered segmentation table that includes every segment id, the exact original text, and the source span metadata.
- The concatenation of all Node A segments in source order must reconstruct the original text without omission. Any gap should halt the run.
- Node B must output a complete open-coding ledger, not a sample. Each record must include the originating segment id, the original segment text, and the assigned initial codes.
- Every segment id generated by Node A must appear in Node B's ledger. Missing, reordered without traceability, or silently merged segments should halt the run.
- Node C and Node D may abstract the data, but they must still retain references back to the originating segment ids or open codes where possible.
- Every execution should expose a stable artifact directory so a researcher can inspect intermediate files directly instead of relying only on the final summary.

## Output Quality Checklist

- The final theory must be explanatory, not merely descriptive.
- The core category must be able to organize the major axial categories.
- The axial-code network should be structured enough to serialize as JSON.
- Important negative cases or contradictions should still appear in the explanation.
- Intermediate artifacts should make it possible to audit how the final theory emerged from the full corpus.
- If the corpus quality is poor, explicitly note the limitation.
